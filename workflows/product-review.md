# 📊 產品審核工作流 (Product Review / CEO Review)

**參與角色**：👔 產品經理 (Product Manager / CEO)
**觸發時機**：在接到全新專案、或是架構師 (Architect) 接手 `spec/` 進行技術設計之前，或是重大功能發佈前。

## 執行步驟：
1. **通盤研讀**：完整讀取 `spec/` 底下的所有功能需求文件。
2. **魔鬼代言人測試**：
   - 使用者為什麼需要這個功能？
   - 這樣設計會不會讓轉換率 (Conversion Rate) 或留存率下降？
   - 有沒有更輕量、更省開發成本的解法可以達到同樣的商業目的？
3. **依賴檢查**：檢查是否有潛在的第三方服務依賴風險（例如使用了某特定行銷工具的 SDK）。
4. **輸出決策**：
   - 將您的優化建議、風險警告或「阻擋放行意見」，輸出為 `spec/PRODUCT_REVIEW.md` 供後續所有角色參考。
   - 若通過則交由 `架構師` 進行下一步的 `build.md` 規劃。
