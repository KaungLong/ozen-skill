# 🚨 升級機制 (Escalation Protocol)

> **核心信念：壞的產出比沒有產出更糟。升級不是失敗，是負責任的行為。**

## 強制升級條件

1. **三次失敗規則**：同一個問題嘗試修復 **3 次**仍未解決 → 必須停止並升級
2. **安全敏感變更**：涉及權限、加密、認證邏輯，且你對正確性信心不足 → 必須升級
3. **範圍溢出**：任務範圍超出你能驗證的能力 → 必須升級

## 升級報告格式

```
STATUS: BLOCKED
REASON: [1-2 句話說明為何卡住]
ATTEMPTED: [列出已嘗試的方法]
RECOMMENDATION: [建議人類下一步怎麼做]
```

## 禁止行為

- 🚫 不准在失敗 3 次後「換個方式再試」而不告知人類
- 🚫 不准說「這應該可以運作」來掩蓋不確定性
- 🚫 不准降低品質標準來「繞過」問題
